# 技术栈

## 项目类型
LeLeTV是一个Web应用程序，包含前端界面和后端服务。它是一个轻量级的在线视频搜索与观看平台原型，主要用于学习现代Web开发技术。

## 核心技术

### 主要编程语言
- **语言**: HTML5, CSS3, JavaScript (ES6+)
- **运行时**: 浏览器环境（前端），Node.js（后端开发服务器）
- **语言特定工具**: npm（包管理），Tailwind CSS（CSS框架）

### 关键依赖/库
- **Tailwind CSS**: 用于构建响应式前端界面
- **HLS.js**: 用于视频流播放
- **Express**: Node.js后端服务器框架
- **Cloudflare Serverless Functions**: 无服务器后端功能实现
- **axios**: HTTP客户端库
- **cors**: 跨域资源共享中间件
- **dotenv**: 环境变量加载工具
- **node-fetch**: Node.js的fetch API实现

### 应用架构
项目采用前后端分离架构：
- **前端**: 静态HTML/CSS/JavaScript文件，使用Tailwind CSS构建响应式界面
- **后端**: Cloudflare Serverless Functions实现API代理，本地开发使用Express服务器
- **数据流**: 前端通过API与后端通信，后端代理第三方视频API请求

### 数据存储
- **主要存储**: 浏览器localStorage（用户设置、观看历史等）
- **缓存**: 浏览器缓存和Service Worker缓存
- **数据格式**: JSON（API通信和本地存储）

### 外部集成
- **APIs**: 第三方视频搜索和播放API
- **协议**: HTTP/HTTPS REST API
- **认证**: 环境变量配置的密码保护

### 监控与仪表板技术
- **仪表板框架**: 原生JavaScript和Tailwind CSS
- **实时通信**: AJAX请求和轮询
- **可视化库**: 原生CSS动画和过渡效果
- **状态管理**: localStorage和内存状态管理

## 开发环境

### 构建与开发工具
- **构建系统**: npm scripts
- **包管理**: npm
- **开发工作流**: nodemon热重载（开发环境）

## 部署与分发
- **目标平台**: Web浏览器（桌面和移动设备）
- **分发方法**: 静态文件部署到Cloudflare Pages或其他静态网站托管服务
- **安装要求**: 现代Web浏览器（支持ES6+）
- **更新机制**: 通过Service Worker实现静默更新

## 技术要求与约束

### 性能要求
- 页面加载时间：3G网络下小于2秒
- 视频播放启动时间：小于3秒
- 内存使用：正常操作期间不超过100MB

### 兼容性要求
- **平台支持**: 主流桌面和移动浏览器（Chrome, Firefox, Safari, Edge）
- **依赖版本**: Node.js v16+，npm最新稳定版
- **标准合规**: HTML5, CSS3, ECMAScript 2015+标准

### 安全与合规
- **安全要求**: 必须设置PASSWORD环境变量，禁止公开部署
- **合规标准**: 遵循OWASP Web安全最佳实践
- **威胁模型**: 主要关注密码保护和API代理安全

### 可扩展性与可靠性
- **预期负载**: 个人使用，低并发用户数
- **可用性要求**: 99%正常运行时间（依赖第三方API可用性）
- **增长预测**: 作为学习项目，不预期大规模用户增长

## 技术决策与原理

### 决策日志
1. **前端框架选择Tailwind CSS**: 选择Tailwind CSS是因为其原子化CSS方法能够快速构建响应式界面，且学习曲线适中，适合学习目的。
2. **后端架构采用无服务器函数**: 使用Cloudflare Serverless Functions是为了学习无服务器架构，同时降低成本和维护复杂性。
3. **视频播放使用HLS.js**: 选择HLS.js是因为其广泛的浏览器兼容性和对HLS流媒体的良好支持。

## 已知限制
- **第三方API依赖**: 项目功能依赖于第三方视频API的可用性，如果API失效会影响功能。
- **浏览器兼容性**: 某些现代JavaScript特性可能在旧浏览器中不支持。
- **本地存储限制**: localStorage有存储大小限制，可能影响大量数据的存储。
