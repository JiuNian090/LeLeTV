以下三项建议**在 Phase 1 中自然消化**，不值得单独投入：

- **StorageService 全量覆盖**：17 文件 130 处直调，仅迁移 12 处。Vite + TS 迁移时会自动暴露未迁移的调用

## Phase 1 — 架构迁移（Vite + TypeScript）

### 目标

将现有纯 JS 脚本体系迁移到 Vite + TypeScript + ES Modules，**零 UI 变化、零功能丢失**。

### 技术栈

| 组件 | 选择 | 原因 |
|------|------|------|
| 构建工具 | Vite 5+ | 极速热更新，按需编译，原生 ESM |
| 语言 | TypeScript 5+ | 渐进式增强，`.js` → `.ts` 逐个文件迁移 |
| 模块系统 | ES Modules (import/export) | Vite 原生支持，Tree Shaking |
| 状态管理 | 极简单例对象（Phase 2 换 Pinia） | 最小依赖，平稳过渡 |
| 事件通信 | 自定义 EventBus（Phase 2 换 mitt） | 最小依赖 |
| 测试 | Vitest | 与 Vite 同构，零配置 |

## Phase 2 — 渐进式引入 Vue

### 目标

在不影响整体功能的前提下，逐步将原生 UI 替换为 Vue 组件，最终完全由 Vue 接管。

### 技术栈

| 组件 | 选择 |
|------|------|
| 框架 | Vue 3（组合式 API + `<script setup>`） |
| 状态管理 | Pinia（替代 Phase 1 的状态单例） |
| 路由 | Vue Router（hash 模式，与现有 hash 路由无缝衔接） |
| 构建 | Vite + @vitejs/plugin-vue（Phase 1 基础上加插件） |
| UI 组件库 | 按需引入 Element Plus 或保持自定义样式 |

### 迁移策略：绞杀者模式

```
现有原生 UI ──→ Vue 组件逐步替换 ──→ 原生 UI 全部移除
     ↑              ↑                       ↑
  保留不变     双架构并存             根节点交由 Vue 管理
```

### 推荐替换顺序

| 顺序 | 组件 | 难度 | 原因 |
|:----:|------|:----:|------|
| 1 | **关于面板** | ⭐ | 纯静态内容，无交互，无外部依赖 |
| 2 | **设置面板** | ⭐⭐ | 独立表单，与主逻辑解耦 |
| 3 | **搜索历史下拉** | ⭐⭐ | 独立 UI 组件，有状态但简单 |
| 4 | **Toast 消息** | ⭐ | 全局通知，无业务逻辑 |
| 5 | **模态框** | ⭐⭐ | 复用组件，替代 `showModal()` |
| 6 | **搜索结果卡片** | ⭐⭐⭐ | 渲染逻辑复杂，但有 data-action 兜底 |
| 7 | **搜索栏** | ⭐⭐⭐ | 核心交互，需谨慎 |
| 8 | **分类（TMDB）** | ⭐⭐⭐ | 独立模块，可完整替换 |
| 9 | **观看历史** | ⭐⭐⭐ | 独立模块 |
| 10 | **导航栏** | ⭐⭐⭐⭐ | 最后替换，因涉及路由整合 |
| 11 | **播放器控制栏** | ⭐⭐⭐⭐ | 最复杂，保留 ArtPlayer 原生控件 |
| 12 | **移除所有原生代码** | ⭐⭐ | 清理收尾 |

### 双架构并存策略

```html
<!-- Vue 接管 #app，原生模块通过 expose 接口交互 -->
<div id="app">
  <!-- Vue 组件逐步替换原生 DOM -->
  <AboutPanel />
  <SettingsPanel />

  <!-- 尚未替换的部分仍由原生 JS 控制 -->
  <div id="legacy-search-results"></div>
</div>
```

```typescript
// 原生模块暴露接口给 Vue
window.LeLeTV.bridge = {
  search(query: string) { /* 调用搜索逻辑 */ },
  playVideo(id: string) { /* 调用播放逻辑 */ },
}
```

## 附录

### A. 迁移风险矩阵

| 风险 | 概率 | 影响 | 缓解措施 |
|------|:----:|:----:|----------|
| `window.fetch` 覆写失效 | 中 | 高 | 改为显式 `apiFetch()`，通过 Vite proxy 开发 |
| 播放器初始化时序错乱 | 低 | 高 | PlayerManager 已封装 destroy/create，加类型保护 |
| data-action 事件委托在模块中失效 | 低 | 中 | 事件委托挂在 `document` 上，不受模块化影响 |
| Tailwind 编译后样式丢失 | 低 | 中 | Vite + Tailwind 插件已验证 |
| Service Worker 缓存旧文件 | 中 | 中 | 版本号 + `npm run build` 自动更新缓存键 |
| Cloudflare Pages Functions 路径变化 | 低 | 高 | functions/ 目录不变，Vite 构建输出到独立目录 |
