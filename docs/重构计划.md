### 关键架构决策

| 决策 | 说明 |
|------|------|
| `window.fetch` 覆写 → `apiFetch()` | 不再覆写全局 fetch，改为显式函数调用（`src/services/api/api.ts`） |
| 共享状态模块 | 新增 `player-state.ts` 解决原 5 个播放器文件的循环依赖 |
| 独立分类 UI 模块 | 新增 `app-category.ts`（16KB），完整 TMDB 分类浏览 UI |
| 事件委托统一点 | 所有 `data-action` 由 `app.ts` + `player.ts` 的 `handleAction` 统一分发 |
| `readyState` 双路径 | 模块加载可能晚于 `DOMContentLoaded`，使用 `readyState` 检查 |

## Phase 2 — 渐进式引入 Vue（待开始）

### 目标

在不影响整体功能的前提下，逐步将原生 UI 替换为 Vue 组件，最终完全由 Vue 接管。

### 技术栈

| 组件 | 选择 |
|------|------|
| 框架 | **Vue 3**（组合式 API + `<script setup>`） |
| 状态管理 | **Pinia**（替代 Phase 1 的状态单例） |
| 路由 | **Vue Router**（hash 模式） |
| 构建 | Vite + @vitejs/plugin-vue（Phase 1 基础上加插件） |

### 迁移策略：绞杀者模式

```
现有原生 UI ──→ Vue 组件逐步替换 ──→ 原生 UI 全部移除
     ↑              ↑                       ↑
  保留不变     双架构并存             根节点交由 Vue 管理
```

### 推荐替换顺序

| 顺序 | 组件 | 难度 | 原因 |
|:----:|------|:----:|------|
| 1 | **关于面板** | ⭐ | 纯静态内容，无交互 |
| 2 | **设置面板** | ⭐⭐ | 独立表单，与主逻辑解耦 |
| 3 | **搜索历史下拉** | ⭐⭐ | 独立 UI 组件 |
| 4 | **Toast 消息** | ⭐ | 全局通知 |
| 5 | **模态框** | ⭐⭐ | 复用组件 |
| 6 | **搜索结果卡片** | ⭐⭐⭐ | 渲染逻辑复杂 |
| 7 | **搜索栏** | ⭐⭐⭐ | 核心交互 |
| 8 | **分类（TMDB）** | ⭐⭐⭐ | 独立模块 |
| 9 | **观看历史** | ⭐⭐⭐ | 独立模块 |
| 10 | **导航栏** | ⭐⭐⭐⭐ | 涉及路由整合 |
| 11 | **播放器控制栏** | ⭐⭐⭐⭐ | 最复杂 |
| 12 | **移除所有原生代码** | ⭐⭐ | 清理收尾 |

## 附录

### A. 迁移风险矩阵

| 风险 | 概率 | 影响 | 缓解措施 |
|------|:----:|:----:|----------|
| `window.fetch` 覆写失效 | 已解决 | 高 | 改为显式 `apiFetch()`，通过 Vite proxy 开发 |
| 播放器初始化时序错乱 | 低 | 高 | PlayerManager 已封装 destroy/create，加类型保护 |
| data-action 事件委托失效 | 已解决 | 中 | 事件委托挂在 `document` 上，`readyState` 双路径 |
| Tailwind 编译后样式丢失 | 低 | 中 | Vite + PostCSS 插件已验证 |
| Service Worker 缓存旧文件 | 中 | 中 | 版本号 + `npm run build` 自动更新缓存键 |
| Cloudflare Pages Functions 路径变化 | 低 | 高 | functions/ 目录不变，Vite 构建输出到独立目录 |

```bash
# Phase 2
npm install vue@latest
npm install -D @vitejs/plugin-vue
npm install pinia vue-router@4
```
