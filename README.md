## 📋 部署指南

### Cloudflare Pages 部署

1. Fork 或克隆本仓库到您的 GitHub 账户
2. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)，进入 Pages 服务
3. 点击"创建项目"，连接您的 GitHub 仓库
4. 使用以下设置：
   - 构建命令：留空（无需构建）
   - 输出目录：留空（默认为根目录）
5. **⚠️ 关键步骤：在"设置" > "环境变量"中添加 `PASSWORD` 变量**
6. （可选）添加 `ADMINPASSWORD` 变量以启用隐藏内容过滤功能的管理权限
7. 点击"保存并部署"

### 本地部署

1. 克隆本仓库到本地
2. 安装依赖：`npm install`
3. 设置环境变量：
   - `PASSWORD`（必填）- 用户访问密码
   - `ADMINPASSWORD`（可选）- 管理员密码，用于管理隐藏内容过滤功能
4. 启动本地服务器：`npm start` 或 `node server.mjs`
5. 在浏览器中访问：`http://localhost:8080`

## ⚙️ 环境变量配置

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `PASSWORD` | 是 | 用户访问密码，必须设置 |
| `ADMINPASSWORD` | 是 | 管理员密码，用于管理隐藏内容过滤功能 |
| `PORT` | 否 | 服务器端口，默认为 8080 |
| `CORS_ORIGIN` | 否 | CORS 允许的源，默认为 * |
| `REQUEST_TIMEOUT` | 否 | 请求超时时间（毫秒），默认为 5000 |
| `MAX_RETRIES` | 否 | 请求最大重试次数，默认为 2 |
| `CACHE_MAX_AGE` | 否 | 静态资源缓存时间，默认为 1d |
| `USER_AGENT` | 否 | 请求 User-Agent，默认为 Chrome |

## 🔧 新功能详解

### 📡 智能负载均衡系统

本项目实现了先进的负载均衡系统，能够智能分配API请求，避免单一源过载，提高整体服务的可用性和响应速度。

**主要特性：**
- **智能API选择**：根据API的健康状态、响应时间、成功率和当前负载动态选择最佳API源
- **健康检查**：定期对所有API源进行健康检查，自动标记不健康的API
- **故障转移**：当某个API源出现问题时，自动切换到其他健康的API源
- **黑名单机制**：对于连续失败的API源，自动加入黑名单并在冷却期后重新评估
- **负载监控**：实时监控每个API源的负载情况，避免过载

**负载均衡状态面板：**
项目还提供了可视化状态面板，可以实时查看各API源的状态信息：
- 健康状态指示
- 成功率统计
- 响应时间监控
- 当前负载显示
- 连续失败次数追踪

### 💾 智能缓存管理系统

为了优化存储空间并保持良好的用户体验，项目实现了智能缓存管理系统。

**主要特性：**
- **自动清理**：定期自动清理过期的临时数据，释放存储空间
- **数据保护**：保留重要的用户设置和历史记录，如选中的API源、观看历史等
- **灵活配置**：可配置清理间隔和数据保留策略
- **手动清理**：提供手动清理缓存的功能，用户可主动释放存储空间

### 🔧 版本管理与更新

项目具备完善的版本管理机制，便于跟踪更新和维护。

**主要特性：**
- **自动版本检测**：自动从CHANGELOG.md中提取最新版本号并在页面中显示
- **语义化版本号**：采用语义化版本控制，格式为 `v{年}.{月}.{日}.{当天提交序号}`
- **更新日志**：完整的更新日志记录，便于了解每个版本的变更内容

## 🛠️ 技术栈

- **前端技术**：HTML5 + CSS3 + JavaScript (ES6+)
- **样式框架**：Tailwind CSS
- **播放器**：ArtPlayer（基于 HLS.js）
- **后端技术**：Node.js + Express
- **代理服务**：服务端 HLS 代理和处理技术
- **部署平台**：Cloudflare Pages / 本地部署
- **其他**：PWA 支持、Service Worker 缓存
- **状态管理**：原生JavaScript类和模块模式
- **本地存储**：localStorage和IndexedDB
- **HTTP客户端**：原生Fetch API

## 🔄 版本管理

项目采用语义化版本控制，版本号格式为 `v{年}.{月}.{日}.{当天提交序号}`。

每次提交代码时，系统会自动从 CHANGELOG.md 中提取最新版本号创建 Git 标签。如果提交信息中包含 `@CHANGELOG.md`，则跳过自动标签创建过程。

可以通过以下方式手动创建标签：

```bash
npm run tag
```
